## 核心问题诊断：为什么找不到 0002、055、021？

### 可能原因清单
#### A. 图标缩放不匹配 ⚠️ (最可能)
- **现象**: 库中图标尺寸 35-67px，但小地图中显示 ~20-40px
- **影响**: 小到库中 80% 版本时，NCC 得分可能显著下降
- **验证方法**: 查看 debug 图像是否有框覆盖目标位置

#### B. 图标被遮挡/部分显示 ⚠️
- **现象**: UI 层叠，某些图标只显示一部分
- **影响**: 特征不完整，NCC 匹配失败
- **例子**: 055 可能被上层图标遮住

#### C. 目标图标不在此截图中 ❓
- **现象**: 用户期望有 0002、055、021，但实际不在小地图范围内
- **验证**: 需要用户手动确认这些图标在 QQ20260412-010841.png 的小地图中

#### D. 特征库构建偏差 🔍
- **055** 是灰色图标 (S=6, V=74)，可能在离线建库时处理不当
- **验证**: 检查 `picture2/icon/055.png` 是否正确加载

```python
# 1. 峰值响应阈值 (目前 0.2)
smooth_resp > 0.2  # 降低 → 更多候选，但更慢；提高 → 更少候选
# 建议范围: 0.1-0.3

# 2. 候选尺寸范围 (目前 20-60px)
for box_size in [20, 25, 30, ..., 60]:  # 扩展范围?

# 3. 色彩检测条件

vivid = (s >= 50 & v >= 65)
grayish = (s < 50 & 40 <= v <= 200)  # 调整 v 范围?
```

```python
# 在 match_icons_in_minimap() 中调整:
```
